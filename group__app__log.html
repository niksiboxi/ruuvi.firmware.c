rd_status_t app_log_init ( )

Initialize logging.

After initialization flash driver is ready to store data. If there is a logging configuration stored to flash, stored configuration is used. If not, default configuration is used and stored to flash.

Return values
RD_SUCCESSif logging was initialized.

rd_status_t app_log_process ( const rd_sensor_data_t *const  sample)

Process data into log.

If time elapsed since last logged element is larger than logging interval, data is stored to RAM buffer. The RAM buffer may be compressed. When the buffer fills STORAGE_BLOCK_SIZE bytes it will be written to flash. If there is no more room for new blocks in flash, oldest flash block is erased and replaced with new data.

Return values
RD_SUCCESSData was logged.
RD_ERROR_NO_MEMORYData cannot be stored to flash and overflow is configured as false.
RD_ERROR_BUSYPrevious operation is in process, e.g. writing to flash.
